Form Views

The form views display common elements of web forms (such as a group of radio buttons, a drop-down list, and a text input). All of these views have a val method that can be used to retrieve their current value. For example: 

splunkjs.mvc.Components.getInstance("example-textinput").val();

Events

All form views have a change event.

change: Fired when the value changes.

Universal Controls.

All form views have an option, disabled, which will disable the view object if passed true, or enable it if passed false.

Radio Group

The Radio Group view represents a radio button group. This view makes it easier to populate a radio button group from a search, but it can also be used like a standard HTML radio button group.

Use the {% radiogroup %} tag and set the default option in the default property:

{% radiogroup id="example-radiogroup" default="One" %}

The radiogroup set is set to:

  • JavaScript
  • Django 
{% radiogroup id="example-radiogroup" default="One"%}

var RadioGroupView = require("splunkjs/mvc/radiogroupview");
new RadioGroupView({
        id: "example-radiogroup",
        default: "One",
        el: $("#divToHangOn")
}).render();

Options

You can set properties for radio button groups, but some can only be set in JavaScript because they take a list that cannot be declared using templates tags.

choices: A static dictionary of options for the radio button group (must be set in JavaScript). 

// Set up choices            
var choices = [
        {label: " One", value: "One"},
        {label:" Two", value: "Two"}, 
        {label:" Three", value: "Three"}];

// Assign them to the button group
splunkjs.mvc.Components.getInstance("example-radiogroup").settings.set("choices", choices);

managerid: The manager that specifies the search to populate the radio button group. For this to be very useful, set valueField, labelField, or both to specify which fields in the search results to use for values and labels. See the example below for more.

valueField: The search field to use for button values.

labelField: The search field to use for button labels. If a labelField is not set, the button labels default to the valueField values.

default: The default selected button.

Events

After a change event, the RadioGroup view returns the value of selected radio button. 

var rbg = splunkjs.mvc.Components.getInstance("example-radiogroup");
var rbResult = document.getElementById("example-radiogroup-result");
rbg.on("change", function(){
        rbResult.innerHTML = rbg.val();
});

Example

In this example, a radio group is created and bound to a search manager that shows all of the available indexes on the Splunk instance. The button group is created in the content block and the search manager is created in the managers block.

{% radiogroup id="example-radiogroup2" managerid="indexes-search" labelField="index" valueField="index"%}

The radio group set is set to:

  • Django

    {% radiogroup id="example-radiogroup2" managerid="indexes-search" labelField="index" valueField="index"%}
{% endblock content %} 

{% block managers %} 
    {% searchmanager id="indexes-search" search="| eventcount summarize=false index=* index=_* | dedup index | fields index" preview=False cache=True %}

Text input

The TextInput view represents a basic text input input. The val method returns the current text.

Use the {% textinput %} tag and set the default text in the default property.

Options

TextInput only has one option:

default: The default text.

Example

In this example, two text inputs are created using templates tags. In JavaScript, when a change event occurs, the text inputs swap their values.

{% textinput id="example-textinput1" default="type here" %} {% textinput id="example-textinput2" default="to be replaced" %}
  • JavaScript
  • Django 

{% textinput id="example-textinput1" default="type here" %} 
{% textinput id="example-textinput2" default="to be replaced" %}

var TextInputView = require("splunkjs/mvc/textinputview");
new TextInputView({
        id: "example-textinput1",
        default: "type here",
        el: $("#divToHangOn")
}).render();

new TextInputView({
        id: "example-textinput2",
        default: "to be replaced",
        el: $("#secondDivToHangOn")
}).render();

How to interact in JavaScript: 

// Get textinput elements
var tb1 = splunkjs.mvc.Components.getInstance("example-textinput1");
var tb2 = splunkjs.mvc.Components.getInstance("example-textinput2");

// Attach change events
tb1.on("change", function(){
    tb2.val(tb1.val());
})
tb2.on("change", function(){
    tb1.val(tb2.val());
})

Passwords

The TextInput can be configured to show an input suitable to passwords and other data that should not be echoed back to the display. The val method returns the current text.

Use the {% textinput %} tag and set the "password" property in the type property.

Single value view

The Single view displays a single value based on a search manager, which can be useful for showing an aggregate value such as a stats count.

Use the {% single %} tag with the managerid property to specify the search manager.

{% single id="example-single" managerid="example-single-search" beforeLabel="Event count:" %}
  • JavaScript
  • Django 
{% block content %}
    {% single id="example-single" managerid="example-single-search" beforeLabel="Event count:" %}
{% endblock content %}

{% block managers %} 
    {% searchmanager id="example-single-search" search="index=_internal | stats count" earliest_time="-15m" latest_time="now" %}   
{% endblock managers %}

var SingleView = require("splunkjs/mvc/singleview");
new SingleView({
    managerid: "example-single-search",
    el: $("#divToHangOn")
}).render();

var SearchManager = require("splunkjs/mvc/searchmanager");
new SearchManager({
    id: "example-single-search",
    search: "index=_internal | stats count",
});

Options

You can set the following properties using templates tags or JavaScript:

managerid: The search manager.

afterLabel: A label to be placed after the value.

beforeLabel: A label to be placed in front of the value.

Field: The search field to use as the display value.

classField: When set to range, works with the rangemap search command to visually convey severity (see the example below). For more on rangemap, see Using rangemap with single value panels.

Example

In this example, the search manager bound to the Single view uses the rangemap search command to create a range field in the search results that indicates a severity of values.
