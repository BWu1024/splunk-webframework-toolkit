The Calendar Heatmap displays time series data on a timeline as a heatmap. It is good for visualizing time-based events and activity and can be configured extensively.

  • require path: splunk_wftoolkit/components/calendarheatmap/calendarheatmap
  • The Calendar Heatmap works best with Splunk searches that use the timechart command
  • Note that the Calendar heatmap will display a different timeline for each field in the results returned from a timechart search

Example Searches

  • index=_internal sourcetype=splunkd_access | head 2000 | timechart span=1m count by status

Options

Option Type Default Explanation Try it
managerid string null The search manager of the chart -
domain string 'hour' The largest unit the heatmap will differentiate by (represented as blocks of squares). Allowed values are hour, day, week, month, year {% dropdown id="domain-dropdown" default="hour" %}
subdomain string 'min' The smaller unit the heatmap differentiates bymin, x_min, hour, x_hour, day, x_day, week, x_week, month, x_month(x_ variants are used to rotate the reading order to left to right, then top to bottom) {% dropdown id="subdomain-dropdown" default="min" %}
options Dictionary {} This dictionary is passed to the wrapped cal heat control as is and sets non-Splunk-specific options. For more on options that can be passed to cal heat see the documentation here: Cal-Heatmap -

Events

Event Properties
click
  • date: the timestamp of the cell clicked
  • value: the value of the field at that time
